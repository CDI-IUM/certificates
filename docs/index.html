How It Works

1. Create Certificates

Use the generator to collect participant data, encrypt it with your secret key, and render a QR code that encodes a verification URL.

2. Share or Print

Download the QR code PNG or share the verification link. Each QR code embeds the encrypted payload.

3. Verify Instantly

The static verification page decodes, decrypts, and validates the payload locally in the browser—no servers required.

4. Audit Anytime

Recipients can validate authenticity even offline thanks to client-side cryptography and cached assets.

Feature Highlights

  • Base64 + XOR encrypted payloads secured by configurable secret key.
  • Responsive verification UI with print mode and accessibility guardrails.
  • Generator workflow with validation, copy-to-clipboard, QR preview, and PNG export.
  • Batch CSV uploads with automated ID assignment and zipped QR downloads.
  • Zero back-end dependencies—perfect for GitHub Pages deployments.

Getting Started Checklist

  1. Update config.json with your real SECRET_KEY and organization metadata.
  2. Deploy this folder to GitHub Pages (or any static host).
  3. Visit generate.html to create certificates and QR codes.
  4. Distribute generated certificates or embed QR codes in your PDF templates.
  5. Share verify.html link so learners can validate their achievement.
